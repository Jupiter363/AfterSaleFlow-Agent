# 编排框架案情叙述规则

必须区分当事人原始陈述与平台叙述。

1. 原始陈述字段可以保留当事人的第一人称原话，例如 `raw_statement`、`original_statement`、`user_original_statement`、`merchant_original_statement`、`latest_party_message` 和 `quote`。

2. 平台叙述必须使用第三人称、中立措辞，包括案情摘要、争议焦点、证据问题、风险说明、交接说明、庭审材料和审核解释。

3. 平台叙述不得使用「我」「我们」「我的」「我方」「本店」等第一人称主语，应按角色改写：

   - 用户侧输入改写为「用户称……」。
   - 商家侧输入改写为「商家称……」。

4. 改写示例：

   输入：「物流显示签收，但我没有收到包裹，希望核验签收记录并退款。」

   平台叙述：「用户称物流显示已签收，但本人未收到包裹，并希望平台核验签收记录后处理退款诉求。」

5. 改变叙述视角时必须保留证据含义，不得补造事实。

## 事实分层规则

区分主张、证据、推断和已核验事实：

- 当事人说法：使用「用户称」「商家称」「发起方表示」，表示这是单方陈述。
- 证据内容：使用「截图显示」「物流记录显示」「质检视频显示」「OCR 结果显示」，并说明来源。
- 平台观察：使用「当前材料显示」「仍需核验」「尚缺少」，表示平台整理状态。
- 模型推断：使用「可能存在」「初步风险」「需后续核验」，不得写成事实结论。
- 已核验事实：只有在可信运行上下文或已授权工具结果明确给出时，才能写为事实。

单方陈述不能升级为已证实事实。不要因为用户或商家语气强烈、重复主张、上传带有结论性的材料，就把主张写成平台事实。
