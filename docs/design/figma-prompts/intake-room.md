# 争议接待室 Figma 提示词

## 产品语境

这是数字人“争议接待官”的会客室。它通过对话判断请求是否构成履约争端，抽取
发起方、订单/售后/物流引用、双方主张、诉求、期望结果和初始风险，输出非最终的
受理建议。

## 视觉基线

延续明亮卡通轻法庭：暖白、天蓝、珊瑚、嫩芽绿、柔和紫；友好的 2D 接待官；
彩色卷宗贴纸和盖章反馈。禁止传统表单瀑布流、黑金法庭和电商客服聊天窗。

## 桌面帧（1440 × 1024）

- 顶部：案件摘要、当前角色、接待官状态、传票信箱。
- 左侧约 38%：有轻微呼吸动画的接待官、对话流和自然语言输入。
- 右侧约 62%：随对话生成的结构化卷宗贴纸：
  - 发起方与关联引用；
  - 用户/商家主张；
  - 诉求与期望结果；
  - 初始风险信号；
  - 缺失信息；
  - 受理建议与置信说明。
- 底部确认区支持纠正字段、确认发起、继续补充。
- `admissible=true` 状态显示“已上报”盖章、相对方传票投递状态和通往证据书房的门。
- `admissible=false` 状态显示留档说明，不出现普通售后入口。

## 角色与状态帧

- USER 发起、MERCHANT 发起各一帧。
- `LISTENING`、`THINKING`、`SPEAKING`、`COMPLETED/HANDOFF`、`ERROR`。
- AI 非最终提示始终可见，不能把受理建议呈现为最终裁决。

## 移动帧

数字人缩为顶部半身卡；卷宗贴纸按“待确认—已确认”折叠；输入与确认动作固定在
底部安全区。
