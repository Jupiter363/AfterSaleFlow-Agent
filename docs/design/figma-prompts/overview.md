# 争议办理总览 Figma 提示词

## 产品语境

AI Native 履约争端审理系统。总览只展示外部接口导入或争议接待官创建的争议订单，
不展示普通订单。用户从这里理解案件旅程并进入当前可处理房间。

## 当前视觉母版

以 Docker 当前运行的 Vue 页面为视觉源头，而不是重新发明一套 Figma 风格。
Figma 的职责是沉淀设计系统、拆分可映射节点、补齐响应式与状态变体。

## 视觉与情绪

- 明亮、活泼、卡通轻法庭乐园，不幼稚。
- 暖白背景，天蓝、珊瑚橙、嫩芽绿、柔和紫作为状态色。
- 使用圆角卷宗卡、印章、信封、路标、小法槌和柔和阴影。
- 2D 数字人支持 `IDLE / LISTENING / THINKING / SPEAKING / COMPLETED /
  HANDOFF / ERROR`。
- 所有 AI 建议明确标注“非最终”；最终落槌属于人类审核员。
- 禁止黑金庄严风、传统电商后台、密集表格、永久侧边栏。

## 桌面帧（1440 × 1024）

- 顶部 88px：品牌“争议游园”、当前角色、搜索、卡通传票信箱与未读数。
- 左侧约 340px：争议订单轨道，卡片显示案件号、订单引用、来源、争议类型、
  当前房间、剩余时效、风险、待处理方和未读数。
- 右侧：可探索的游园路线图，节点依次为接待室、证据书房、小法庭、平台终审、
  结果。路线展示已完成、当前、等待、锁定和超时状态。
- 右下：当前数字人提示、下一动作、“进入当前房间”主按钮。
- 默认数据至少含外部导入、接待官创建、举证中、庭审中、待终审和已完成案件。
- 点击左侧案件只切换右侧旅程；点击主按钮或可用节点才导航。

## 移动帧（390 × 844）

- 顶部保留品牌、角色和信箱。
- 订单轨道改为横向卡片滑轨。
- 路线图改为纵向游园路径，当前节点始终首屏可见。
- 主动作固定在安全区上方，不能遮挡倒计时和状态说明。

## 可访问性

颜色之外同时使用图形、文字和状态标签；键盘焦点清晰；动画提供
`prefers-reduced-motion` 替代。
